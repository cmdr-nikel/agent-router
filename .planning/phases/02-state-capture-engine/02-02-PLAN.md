| dspy LM call → TrajectoryCallback | LM output text and usage cross into captured telemetry (in-process; no network, no external sink) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | callback storing live `inputs` dict by reference, mutating the in-flight call | mitigate | Never store the inputs dict by reference (Pitfall P5); capture scalars only — this callback records no raw input dict |
| T-02-04 | DoS | `_pending_lm` grows unbounded if on_lm_end never fires (crash before callback) | mitigate | `_pending_lm.pop(call_id, None)` in on_lm_end; dict is per-callback (per-session, short-lived) so worst-case bound is one tracked run |
| T-02-05 | Info disclosure | TurnRecord.output_text holds raw LM output (may contain prompt-derived secrets/PII) | accept | Library stores what DSPy produces; downstream logging/PII handling is the caller's responsibility — documented in capture.py docstring (RESEARCH §Security Domain) |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages (RESEARCH §Package Legitimacy Audit: none) |
</threat_model>
