## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer call site → TrajectoryTracker | caller supplies session_id; tracker mutates global dspy callback settings (scoped) and the shared _SESSION_REGISTRY |
| concurrent trackers → _SESSION_REGISTRY | multiple sessions share the registry dict; the boundary is _REGISTRY_LOCK + per-session _lock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | dspy.settings.configure used by mistake, clobbering user's Langfuse/W&B callbacks | mitigate | Use dspy.context(callbacks=existing+[cb]) exclusively (D-01 / Pitfall P4); CAP-02 test asserts preservation |
| T-02-07 | DoS | _SESSION_REGISTRY unbounded growth across many tracked runs | mitigate | __exit__ pops session_id under _REGISTRY_LOCK (closes Phase-1 TODO); eviction asserted post-exit |
| T-02-08 | Tampering | TOCTOU race on registry check-then-insert under concurrency | mitigate | check-then-insert held under _REGISTRY_LOCK (Phase-1 CR-01 fix) |
| T-02-09 | Info disclosure | caller-supplied session_id used as registry key (no validation) | accept | In-process library, no external surface (RESEARCH §Security Domain V3 partial); session_id non-empty is a documented caller expectation, not enforced in Phase 2 |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new packages (RESEARCH §Package Legitimacy Audit: none) |
